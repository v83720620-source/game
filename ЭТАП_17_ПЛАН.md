## 🎯 ЭТАП 17.1: LISTEN SERVER SETUP

### **Что такое Listen Server?**

```
Listen Server = Host + Client в одном
├── Один игрок = Server
├── Другие подключаются к нему
├── Бесплатно
├── Простая настройка
└── ⚠️ Host advantage (host имеет 0ms ping)

Альтернатива: Dedicated Server
├── Отдельная машина = Server
├── Все игроки = Clients
├── Платно (хостинг)
└── ✅ Fair play (все равные ping)
```

### **Рекомендация Unity 2026:**

```
Для мобильных FPS игр:
✅ Listen Server для начала (P2P)
✅ Unity Relay для NAT traversal
✅ Matchmaking через Unity Services

Для релиза:
✅ Dedicated Server для ranked modes
✅ Listen Server для casual/custom games
```

### **17.1.1 Unity Relay Integration (2 дня)**

**Что это:**
```
Unity Relay = P2P connection сервис
├── Обходит NAT/Firewall
├── Не нужен port forwarding
├── Работает за любым роутером
├── До 100 concurrent users бесплатно
└── Интеграция с Netcode

Без Relay:
❌ Нужен port forwarding (сложно)
❌ Не работает за некоторыми NAT
❌ Проблемы с firewall

С Relay:
✅ Работает везде
✅ Автоматическое подключение
✅ Бесплатно для инди
```

**Что создадим:**
```csharp
Scripts:
├── RelayManager.cs
├── RelayHostManager.cs
├── RelayClientManager.cs
└── RelayUI.cs

Features:
✅ Create Relay allocation
✅ Generate join code
✅ Join by code
✅ Host/Client connection
✅ Error handling
```

## ⚡ ЭТАП 17.3: OPTIMIZATION

### **17.3.1 Network Optimization (2 дня)**

**Reduce bandwidth:**
```csharp
Techniques:
✅ Delta compression (only send changes)
✅ Update rate reduction (10Hz for non-critical)
✅ Interest management (только nearby players)
✅ Network LOD (дальние players → low freq)
✅ Quantization (position: 0.01m precision)

Implementation:
├── NetworkTransform settings
├── NetworkVariable update rates
└── Custom serialization
```
